# Установка FinanceTracker Desktop на ПК

## Текущая версия: v1.0.6 (sql.js)

Приложение использует **sql.js** (WebAssembly SQLite) — не требует нативных модулей, совместимо с любым ПК.

## Варианты установки

### Вариант A: Portable версия (рекомендуется, без установки)

1. Скачайте папку `dist-electron/win-unpacked/` из репозитория
2. Скопируйте в любую папку (например, `C:\FinanceTracker\`)
3. Запустите `FinanceTracker.exe`
4. Создайте ярлык на рабочем столе (правой кнопкой → "Создать ярлык")

### Вариант B: Графический установщик (рекомендуется)

1. Запустите `installer/run-installer.bat` **от имени администратора**
2. Откроется красивое окно установщика с информацией о программе
3. Нажмите **«Далее»** → выберите путь установки → **«Установить»**
4. После завершения нажмите **«Запустить»** для старта приложения

**Требования**: Windows 7+, PowerShell 5.0+

### Вариант C: Установщик Windows (NSIS)

1. Скачайте `dist-electron/FinanceTracker Setup 1.0.6.exe`
2. Запустите установщик от имени администратора
3. Следуйте инструкциям мастера установки

### Вариант D: Скрипт установки

1. Запустите `SimpleInstall.bat` от имени администратора
2. Скрипт автоматически:
   - Создаст папку `C:\FinanceTracker\`
   - Скопирует файлы
   - Создаст ярлык на рабочем столе

## Технические детали

### Почему sql.js вместо better-sqlite3?

| Параметр                 | better-sqlite3                     | sql.js                           |
| ------------------------ | ---------------------------------- | -------------------------------- |
| Тип                      | Нативный модуль (C++)              | WebAssembly                      |
| ABI-зависимость          | Да (компилируется под Node.js ABI) | Нет                              |
| Совместимость с Electron | Требует `@electron/rebuild`        | Работает из коробки              |
| Размер                   | ~5 МБ                              | ~1.5 МБ (WASM)                   |
| Производительность       | Высокая                            | Средняя (достаточно для desktop) |

### WASM файл

sql.js загружает `sql-wasm.wasm` из `app.asar.unpacked`. Файл автоматически распаковывается при сборке благодаря конфигурации `asarUnpack` в `package.json`.

## Диагностика проблем

### Шаг 1: Проверка логов

Логи приложения находятся в:

```
%APPDATA%\FinanceTracker\logs\
```

Проверьте файлы `main.log` и `error.log`

### Шаг 2: Запуск с отладкой

```cmd
cd C:\FinanceTracker
FinanceTracker.exe --enable-logging --v=1
```

### Шаг 3: Проверка WASM файла

Убедитесь, что файл `sql-wasm.wasm` существует:

```
dir "C:\FinanceTracker\resources\app.asar.unpacked\node_modules\sql.js\dist\sql-wasm.wasm"
```

## Частые ошибки и решения

### Ошибка: "A JavaScript error occurred in the main process"

**Причина**: Ошибка в коде main process (обычно видна в логах)
**Решение**:

1. Проверьте `%APPDATA%\FinanceTracker\logs\error.log`
2. Убедитесь, что БД не повреждена: удалите `%APPDATA%\FinanceTracker\finance-tracker.db`
3. Перезапустите приложение

### Ошибка: "The application was unable to start correctly (0xc000007b)"

**Причина**: Проблема с архитектурой (x86 vs x64) или поврежденные DLL
**Решение**:

1. Установите Visual C++ Redistributable
2. Запустите `sfc /scannow` в командной строке от администратора
3. Используйте версию для вашей архитектуры (x64 для 64-bit Windows)

### Ошибка: "Failed to execute script"

**Причина**: Блокировка исполняемого файла
**Решение**:

1. Скачайте архив заново
2. Распакуйте с помощью 7-Zip или WinRAR
3. Проверьте, не заблокирован ли файл (свойства → "Разблокировать")

## Антивирусная защита

### Windows Defender

1. Откройте "Безопасность Windows" → "Защита от вирусов и угроз"
2. Нажмите "Управление настройками" → "Исключения"
3. Добавьте исключение для папки с приложением

### Другие антивирусы

- Добавьте папку с приложением в исключения
- Временно отключите антивирус на время установки

## Техническая поддержка

### Для получения помощи:

1. **Предоставьте информацию**:
   - Версию Windows (`winver`)
   - Скриншот ошибки
   - Содержимое логов из `%APPDATA%\FinanceTracker\logs\`
   - Сообщение антивируса (если есть)

2. **Проверьте**:
   - Достаточно ли свободного места на диске
   - Не запущены ли другие копии приложения
   - Не блокирует ли брандмауэр

3. **Попробуйте**:
   - Перезагрузить компьютер
   - Установить в другую папку
   - Использовать другую учетную запись пользователя
