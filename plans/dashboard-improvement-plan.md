# План улучшения Dashboard

## Текущие проблемы

- Метрики считаются за всё время, а не за текущий месяц
- Нет прогресса по бюджетам и целям
- Нет предстоящих регулярных платежей
- Нет быстрых действий
- Нет визуализации расходов по категориям
- Пустое состояние скудное

## План работ

### Шаг 1: Создать хук `useDashboardData`

Вынести всю логику расчётов в отдельный хук для чистоты компонента.

**Файл:** `src/hooks/useDashboardData.ts`

**Что делает:**

- Фильтрует транзакции за текущий месяц
- Считает доходы/расходы/баланс за месяц
- Считает "свободно" (доходы - расходы - бюджеты)
- Сортирует бюджеты по проценту использования (топ-3)
- Сортирует цели по прогрессу (топ-3)
- Находит ближайшие регулярные платежи (следующие 5)
- Группирует расходы по категориям для диаграммы

### Шаг 2: Создать компонент `MetricsGrid`

**Файл:** `src/components/dashboard/MetricsGrid.tsx`

4 карточки:

1. **Доходы за месяц** — зелёный, иконка TrendingUp
2. **Расходы за месяц** — красный, иконка TrendingDown
3. **Баланс** — синий/оранжевый, иконка Wallet
4. **Свободно** — бирюзовый, иконка PiggyBank (новая метрика: доход - расход - бюджеты)

Каждая карточка — `Card` с иконкой, заголовком и суммой.

### Шаг 3: Создать компонент `BudgetProgress`

**Файл:** `src/components/dashboard/BudgetProgress.tsx`

- Показывает топ-3 бюджета с наибольшим процентом использования
- Прогресс-бар с цветом (зелёный < 70%, жёлтый 70-90%, красный > 90%)
- Если бюджетов нет — кнопка "Создать бюджет"
- Если бюджетов > 3 — ссылка "Все бюджеты"
